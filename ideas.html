Possible things to try and do with the code

Some suggestions will be simple and straight forward, while others are quite advanced projects. The following have been sorted by topic - this grouping does not reflect how easy or hard each will be to achieve...
  • Enhancements to existing games:
    There is an issue in Chrome where the 'alert' box can be displayed ahead of and block the rendering of 'document' changes done prior to invoking the 'alert'. This is a known issue https://stackoverflow.com/questions/52953826/chrome-javascript-execution-sequence-problem-with-alert. Some of the games here still suffer from this, so could change the code to use 'setTimeout' to delay the 'alert' sufficiently to allow the rendering of 'document' changes. Using a reasonable time duration should not adversely affect the user experience.
    • Maze
      1. Make it timmed
      2. Have a complexity setting
      3. Have a local map size setting
      4. Have objects in the maze - could do this using Windings font of different sizes so the same character displays at different sizes depending on your distance from it
      5. Ability to indicate which paths you have already searched - could do this either with breadcrumbs or by writing on the walls
      6. Have ability to modify the maze by breaking through or building walls
      7. Be chased through the maze
    • Tiles
      1. Use CSS sprite image to programmatically make tiles from a single complete image
      2. Ability to load your own picture for the tiles (would need the CSS sprite change to be done first)
      3. Ability to define the number of rows and columns (would need the CSS sprite change to be done first)
      4. If you randomly position the tiles rather than randomly do moves, will you always be able to solve the puzzle? If not, can you find a better algorithm for creating a random configuration to solve than performing moves?
    • Hang Man
      1. Add more words
      2. Try and connect to an online random word generator
      3. Inplement difficulty settings (how do you determine how difficult a word is - consider number of different letters - consider how frequently the word is used)
    • Minefield
      1. Implement right click to 'flag' a mine which would prevent you accidently clicking on that square
      2. Find objects in the mine field - could you find power ups of extra lifes or a single use mine detector
  • Additional new games:
    1. CSS may provide ways to rotate images so may be able to write a jigsaw game - can rotate pieces in multiples of 90 degrees, and move pieces. Need to design how to attach and detach pieces.
  • Include games which require player logic:
    1. Dr Nim
    2. Tic-Tac-Toe (noughts and crosses)
    3. Reversi/Othello
  • Other apps to write which are not necessarily games:
    1. Morse Code translator
    2. Random dot stereogram generator
